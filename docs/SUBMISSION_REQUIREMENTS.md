# Submission requirements

## Team feedback (mandatory)

All team members must submit Bob usage via the feedback page, completing all mandatory fields.

Feedback submission is required before project submission is enabled and unlocks access to the GitHub repository. Teams where no members — or only a subset of members — submit feedback will be disqualified.

---

## Project submission

By end of day, each team must zip and post the following materials to the official Bob-A-Thon repo.

### Demo video

- Maximum 5 minutes
- Demonstrates the solution your team built

### Prompt documentation

- Shows the prompts used
- Explains the usage approach
- Provides examples of Bob utilization

### Problem statement

- Brief explanation of the problem
- Why solving it matters
- Proposed solution overview
- Use diagrams via Plan mode

### GitHub repository

- Code must be properly readable and executable
- Working implementation required
